## Conventions (follow strictly)

### Python
- Package manager: `uv`. Define all dependencies in `pyproject.toml`. Never use bare `pip install`.
- Run scripts with `uv run python -m src.cli <args>`.
- Use `src/` layout with `__init__.py` files.
- After editing Python files run: `black .` then `ruff check --fix .`

### LaTeX
- Place a `.latexmkrc` in the repo root that sets XeLaTeX + biber, outputs aux files to `_latexmk/`, and writes the final PDF beside the `.tex` file.
- **Always `cd` to the directory containing the `.tex` file before compiling**, then run `latexmk <filename>` with no extra arguments.
- Modularise every document: one `\input{sections/foo.tex}` per logical section.

### Figures
- Generate all figures as PNG from Python scripts in the document's `figs/` directory.
- Use `graphviz.Digraph` with `fontname="Helvetica"`, `fontsize="10"`, `dpi="300"`, `cleanup=True`.
- Use colored subgraph clusters. One script per figure. Script writes PNG to its own directory.

### References
- All `.bib` entries must be real, web-verified publications (check DOI or URL resolves).
- Use `biblatex` with `style=ieee` and `backend=biber`.
- Include at least 15 references spanning dataset papers, ICS/SCADA security, Windows Event Log analysis, and the AI method you choose.

## Deliverable 2 ‚Äî Python CLI (`src/` + `README.md`)

### Synthetic Data Generator (`src/data/generator.py`)

Generate realistic Windows Event Log sequences for a smart grid OT/IT environment. The generator must:

- Produce structured records with fields: `timestamp`, `event_id`, `source`, `user`, `hostname`, `description`, `label` (0=benign, 1=malicious).
- Cover standard event IDs relevant to the domain:
  - Authentication: 4624 (logon), 4625 (failed logon), 4634 (logoff), 4648 (explicit credentials)
  - Process: 4688 (process creation), 4689 (process termination)
  - Privilege/account: 4672 (special privileges), 4720 (account created), 4732 (group membership)
  - Service: 7045 (service installed), 7036 (service state change)
  - Object access: 4663 (object access attempt), 4656 (object handle requested)
- Simulate smart grid-specific hostnames/users (e.g., `SCADA-HMI-01`, `RTU-CTRL-03`, `eng_station`).
- Generate benign sequences (normal operator patterns, scheduled tasks) and malicious sequences covering at least 4 attack patterns:
  1. **Lateral movement**: repeated 4625 failures ‚Üí 4624 success from unexpected source
  2. **Persistence**: 7045 new service install on SCADA host
  3. **Privilege escalation**: 4672 + 4688 sequence with unusual process
  4. **Reconnaissance**: burst of 4663 object access on HMI/RTU hosts
- Configurable: `--n-benign`, `--n-malicious`, `--seed`, `--output` path.
- Output: CSV with one row per event; sequences identifiable by `session_id`.

### Feature Engineering (`src/features/engineer.py`)

Extract session-level (or sliding-window) feature vectors from raw event sequences:
- Event frequency counts per event ID
- Time-delta statistics (mean, std, min, max inter-event gap)
- Unique source/user/host counts per session
- Rare event flags (events appearing in <5% of benign sessions)
- Sequence entropy (Shannon entropy over event ID distribution)

### Model (`src/models/train.py`)

Choose the AI method you judge most appropriate for this structured tabular problem (justify your choice in a comment at the top of the file). Requirements:
- Scikit-learn compatible API or PyTorch.
- 5-fold stratified cross-validation.
- Hyperparameter search (grid or random).
- Save the best model as `models/best_model.pkl` (or `.pt`).

### Evaluation (`src/models/evaluate.py`)

Report on held-out test set:
- Accuracy, Precision, Recall, F1 (macro + per-class)
- ROC-AUC
- Confusion matrix (saved as PNG to `models/confusion_matrix.png`)

### CLI (`src/cli.py`)

```
uv run python -m src.cli generate --n-benign 5000 --n-malicious 1000 --output data/raw/logs.csv
uv run python -m src.cli preprocess --input data/raw/logs.csv --output data/processed/
uv run python -m src.cli train --data data/processed/ --output models/
uv run python -m src.cli evaluate --model models/best_model.pkl --data data/processed/test.csv
```

### Beamer slides (`presentation.tex`)

- ~15‚Äì18 slides covering: title, agenda, problem motivation, related work (2 slides), proposed approach, data & features, model, results, discussion, conclusion, references, Q&A.

### Teleprompter (`teleprompter.tex`)
- Standalone LaTeX document (`article` class, large font ‚â•14pt for readability).
- One section per slide, numbered to match slide numbers (e.g., `\section*{Slide 3 ‚Äî Related Work}`).
- Each section contains the full spoken script for that slide: complete sentences, natural speech rhythm, ~60‚Äì90 seconds of content per slide.
- Include delivery cues in italics: *[pause]*, *[click to next bullet]*, *[show figure]*.
- A header on every page showing the slide number and title for quick navigation.
- Compile independently: `cd docs/presentation && latexmk teleprompter.tex`
